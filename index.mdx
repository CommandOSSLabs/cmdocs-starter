# Welcome

You just scaffolded a brand-new documentation site with `cmdocs init`. Everything you see is a starting point — open the files and make them yours.

<Callout type="info">
  Your dev server is (or is about to be) running at [http://localhost:3000](http://localhost:3000). Every save to `docs.json`, an `.mdx` file, or a logo hot-reloads the browser.
</Callout>

## First-edit checklist

Work through this list before you add content of your own — each item takes under a minute.

<Steps>
  <Step>
    ### Rename your site

    Open `docs.json` and change `name`, `navbar.title`, `seo.title`, and `footer.copyright` to your own. These flow into the browser tab title, navbar, and SEO meta tags.
  </Step>
  <Step>
    ### Replace the logos

    Drop your own SVG into `public/logo/light.svg` and `public/logo/dark.svg`. Keep the filenames or update `navbar.logo` in `docs.json` to point at new ones. Also replace `public/favicon.svg`.
  </Step>
  <Step>
    ### Point GitHub at your repo

    In `docs.json`, swap `navbar.github` to your project's GitHub URL — the navbar icon will link there.
  </Step>
  <Step>
    ### Edit this page

    Open `index.mdx` and replace this checklist with your project's real introduction.
  </Step>
  <Step>
    ### Validate before you commit

    Run `npx cmdocs check` — it verifies `docs.json`, MDX frontmatter, referenced assets, and every internal link. Wire it into a git pre-commit hook so broken docs never reach `main`. See the [Validate guide](/documentation/guides/validate).
  </Step>
  <Step>
    ### Deploy

    Push your repo to GitHub and connect it in the [cmdocs dashboard](https://cmdocs.sh). Every future push triggers a build and deploys to `<your-project>.cmdocs.app`. See the [Deploy guide](/documentation/guides/deploy).
  </Step>
</Steps>
